# 微信公众号"XX天系列"教程写作指南

本 skill 帮你快速写出结构规范、**内容经查证准确**的系列教程文章。

## 核心原则：查证优先，禁止胡编

**所有技术内容必须查证**：
- ❌ 禁止：凭记忆写 API 参数名、版本号、函数签名
- ✅ 必须：查官网文档或用 context7 MCP 确认

**查证优先级**：
1. **context7 MCP**（推荐）：`mcp__plugin_context7_context7__search-docs` 或 `mcp__plugin_context7_context7__query-docs`
2. **官网文档**：LangChain→python.langchain.com，Docker→docs.docker.com 等
3. **MCP 工具**：先尝试 context7，失败再用 WebSearch
4. **禁止**：直接用训练数据中的知识，不确定就查

## 工作流程

### 第一步：确认系列主题和大纲

与用户确认：
1. **主题**：要教什么？（如 Docker、Kubernetes、FastAPI）
2. **天数**：分几天讲？（建议 5-14 天）
3. **目标读者**：初学者 / 有一定基础 / 面试冲刺
4. **核心价值**：学完能解决什么问题？

然后设计每天的主题分配，确保：
- 第 1 天：基础概念 + 环境搭建
- 中间几天：核心功能层层递进
- 最后一天：总结 + 面试题 / 实战项目

### 第二步：创建项目结构

```
langchain-xxdays/
├── api/                    # 代码示例目录
│   ├── day1_xxx.py
│   ├── day2_xxx.py
│   └── ...
├── tests/
│   └── test_article_accuracy.py  # 验证代码准确性
└── pyproject.toml          # 依赖管理
```

使用 `uv` 管理项目依赖。

### 第三步：内容查证（必须执行）

每写一个技术知识点前，先查证：

**1. 版本号和最新 API**
- 使用 `context7` 查询最新文档：`mcp__plugin_context7_context7__search-docs`
- 示例：`context7 query "LangChain create_agent API parameters"`
- 确认：函数名、参数名、返回值结构

**2. Import 路径**
- 用 `uv run python -c "from xxx import yyy"` 验证
- 示例：`uv run python -c "from langchain.agents import create_agent"`
- **不确定路径时，必须验证，不能猜**

**3. 代码示例**
- 先写最小可运行示例
- 在项目目录运行验证
- 有报错必须修复，不能忽略

**4. 相关链接**
- 必须提供真实的官方文档 URL
- 禁止使用占位符或过期链接

### 第四步：写作格式规范

每篇文章必须包含：

#### 1. 标题格式
```
# [主题] N 天系列 Day N｜[核心价值一句话]
```

#### 2. 开篇引言（痛点 + 价值）
```markdown
> 痛点描述（不超过3行）
> 学完后的收益
```

#### 3. 章节结构
```
## 一、[主题块]
## 二、[主题块]
...
---
## 相关链接
```
- 章节用 `---` 分隔
- 汉字编号：一、二、三...
- 核心能力用 ✅ 列表
- 实战案例用引用块

#### 4. 架构图（ASCII）
用 ASCII 字符画架构图，帮助读者建立全局认知：
```
┌─────────────────────────────────────┐
│              标题                    │
│  ┌─────────┐    ┌─────────┐         │
│  │  组件1  │───▶│  组件2  │         │
│  └─────────┘    └─────────┘         │
└─────────────────────────────────────┘
```

#### 5. 代码示例
- 代码块配简短中文注释
- 关键行用 `# 注释` 标注
- 代码必须可运行

#### 6. 表格对比
用于对比概念/方案：
```markdown
| 维度 | 方案A | 方案B |
|------|-------|-------|
| 优点 | ...   | ...   |
| 缺点 | ...   | ...   |
```

#### 7. Day 回顾表格
```markdown
| 知识点 | 说明 |
|--------|------|
| ...    | ...  |
```

#### 8. 结尾相关链接
```markdown
---

## 相关链接

微信内长按复制链接到浏览器打开。

```
官方文档：https://example.com/
GitHub：https://github.com/example
```
```

### 第五步：代码准确性验证

写完文章后，验证代码：

1. **Import 路径必须正确**：
   - 使用 `uv run python -c "from xxx import yyy"` 验证每个 import
   - 不确定路径时，先查文档或用 `uv run python -c "import pkg; print(pkg.__file__)"`

2. **API 调用签名正确**：
   - 使用 `inspect.signature()` 检查函数参数
   - 实例化验证构造函数接受正确参数

3. **运行测试**：
   ```bash
   uv run python tests/test_article_accuracy.py
   ```

## 文章命名规范

文件名前缀用日期：
```
YYYYMMDD-[主题]-DayN-[副标题].md
```

示例：
- `20260420-LangChain-7天-Day1-5分钟搞懂四大组件.md`
- `20260421-Docker-14天-Day1-5分钟搞懂容器核心概念.md`

## 代码目录规范

每个 Day 对应一个 `.py` 文件：
```
api/day1_components.py   # Day 1 代码
api/day2_model_io.py     # Day 2 代码
```

## 写作风格要点

| 要点 | 示例 |
|------|------|
| 短句为主 | ✅ "AI 每次对话都是新手" ❌ "由于 AI 没有记忆能力，因此在每次新的对话中" |
| 量化收益 | ✅ "Token 消耗降低 60%" ❌ "大幅降低 Token 消耗" |
| 动宾短语 | ✅ "搞懂概念" ❌ "对概念的深入理解" |
| 具体场景 | ✅ "查天气、搜航班" ❌ "调用外部 API" |

## 禁忌清单

- ❌ **胡编乱造 API 参数名、版本号、函数签名**
- ❌ 模板化的"本文价值"列表
- ❌ "担心...？想...？"引导句
- ❌ "这是**唯一需要提前准备**的内容"等废话
- ❌ 「如有疑问欢迎留言」等无效客套
- ❌ 「总的来说」「综上所述」等套话
- ❌ 无实战支撑的效率数据
- ❌ 使用占位符 URL 或过期链接

## 输出清单

每个 Day 完成时，确保输出：

1. **文章文件**：`articles/YYYYMMDD-主题-DayN-标题.md`
2. **代码文件**：`langchain-xxdays/api/dayN_name.py`
3. **验证结果**：运行 `uv run python tests/test_article_accuracy.py` **必须无报错**
4. **查证记录**：关键技术点需注明来源（如"来源：LangChain 官方文档"）

## 查证清单（每篇必备）

写完文章后逐项检查：

- [ ] 所有 import 路径已验证可用
- [ ] 所有 API 函数签名已查证
- [ ] 版本号已查证（如 LangChain 1.2.15）
- [ ] 相关链接均为有效官方 URL
- [ ] 代码示例已运行验证
- [ ] 无"应该是这样"类未经确认的表述
